Section 1: Purpose and Definitions

Margin and padding concepts are utilised in order to:

  • Create/remove the space between "outsides" of different html elements (margin);
  • Create/remove the space within elements (padding).
The "Box Philosophy"

One of my teachers had introduced my to a concept of "Box Phillosophy". The cornerstone of this doctrine is that everything in html (and maybe in life) is a box. Some boxes take the whole length of their parent elements (block-elements), some share line-space with neighbors (inline-elements). Some elements can take the whole screen! Margins and padding allow us to create and adjust the space between boxes. An absolutely cosmic concept, to be honest.

Another important thing to remember, when things get a bit overwhelming emotionally, is that a chocolate box is a box too and so is the coffin.

Code Examples

The size of the margin area is determined by the following properties that you can add in your css file:

  • "separate" properties margin-top, margin-right, margin-bottom, margin-left;
  • shorthand property margin that incapsulates all the above.

You can use margin shorthand property by specifying all four dimensions separated by a comma by typing either four values or three or two or one. Examples are below:

Specifying top margin, left, right, bottom.

element {
margin: 1px 2px 1px 2px;
}

Specifying top margin, left&right, bottom.

element {
margin: 1px auto 2px;
}

Specifying top&bottom, left&right margins.

element {
margin: 1px auto;
}

Specifying all four margins.

element {
margin: 1px;
}

Same applies for padding.

Section 2: Margin vs Padding

Another teacher of mine recommends to be consistent above all. If we you are using margins then stick to it, if it's padding which is your weapon of choice, then love it. Also, the same instructor had mentioned, that if working for or with someone, different web-developers often might have their own little strange preferences favouring either margins or padding.

What I have learned after doing a couple of practice runs with certifications projects, mixing two together can make the document very messy, like Red Wedding from GoT. Pesronally, I prefer my layouts like I prefer my scotch: neat.

Section 3: Tips and Tricks

Following the "Box Philosophy" it would be nice to see the actual boxes to better understand and identify margins, padding and just in general. I found enabling borders for all elements is very helpfull. This can easily be done with adding the following code in your CSS file (and removing later when it's not needed anymore):

* {
border: 1px solid black;
}

Makes a huge diffrence.
